## What it does

Every feature the editor has, exposed as a clean REST API.

- **Templates** — CRUD, publish, clone, render to HTML.
- **Media** — Upload, browse, organize, delete.
- **Snapshots** — List, compare, restore.
- **Rendering** — Compile any template to MJML or HTML via API. Supports merge-tag substitution.
- **Webhooks** — Subscribe to template events (created, updated, published).

## Why it matters

Cloud is headless-first. The visual editor is one client; your dashboards, automations, and integrations are others. Nothing is locked behind the UI.

## Good fits

- Building custom dashboards on top of Cloud data
- Automating template production from a CMS or CRM
- Server-side rendering templates into transactional email flows
